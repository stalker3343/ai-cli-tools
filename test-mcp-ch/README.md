# Монитор записи к врачу — er.center-zdorovie.ru

Скрипт проверяет расписание врача каждые 30 минут и уведомляет, когда появляется свободный слот.

---

## Быстрый запуск

```bash
# Установить зависимости (один раз)
pip install requests

# Запустить из папки со скриптом
python felitsin_checker.py
```

Лог пишется в `felitsin_log.txt` рядом со скриптом.  
Остановить: `Ctrl+C`.

---

## Что нужно перед первым запуском

Скрипту нужны куки от авторизованной сессии. Их надо вставить **один раз**.

### Как получить куки

1. Открыть Chrome, зайти на [er.center-zdorovie.ru](https://er.center-zdorovie.ru) и войти в личный кабинет.
2. Нажать `F12` → вкладка **Network**.
3. Обновить страницу или нажать на любую кнопку на сайте.
4. В списке запросов найти любой к домену `er.center-zdorovie.ru` → кликнуть на него.
5. В разделе **Request Headers** найти строку `Cookie:`.
6. Скопировать значения трёх кук:
   - `PLAY_SESSION`
   - `WR_SESSION`
   - `WR_DETAIL`
7. Вставить их в скрипт в словарь `COOKIES`.

### Как долго действуют куки

Пока скрипт работает — **бесконечно**. Сервер продлевает сессию с каждым ответом через `set-cookie`, и скрипт автоматически обновляет куки в памяти. Вручную ничего делать не нужно.

Если скрипт был **остановлен дольше ~2–4 часов** — куки протухнут. Скрипт напишет `СЕССИЯ ИСТЕКЛА` — нужно зайти на сайт заново, скопировать свежие куки и перезапустить.

## Как настроить для другого врача

Открыть скрипт и изменить параметры в блоке `# Настройки`:

```python
DOCTOR_ID   = "990005142"   # ID врача
FILIAL_ID   = "4"           # ID филиала
SPECLIST    = "1031"        # ID специальности
WEEKS_AHEAD = 8             # сколько недель вперёд смотреть
CHECK_EVERY = 1800          # интервал проверки в секундах (1800 = 30 мин)
```

### Как найти нужные ID

1. Открыть Chrome → F12 → Network → вкладка **Fetch/XHR**.
2. На сайте открыть расписание нужного врача.
3. Найти запрос вида:
   ```
   GET /api/reservation/schedule?st=...&en=...&doctor=XXX&filialId=YYY&speclist=ZZZ
   ```
4. Из URL скопировать:
   - `doctor=` → `DOCTOR_ID`
   - `filialId=` → `FILIAL_ID`
   - `speclist=` → `SPECLIST`

Также из этого запроса удобно взять свежие куки (вкладка **Request Headers → Cookie**).

## Как устроен API

### Эндпоинт расписания

```
GET https://er.center-zdorovie.ru/api/reservation/schedule
```

**Параметры:**

| Параметр    | Пример       | Описание                              |
|-------------|--------------|---------------------------------------|
| `st`        | `20260427`   | Начало недели (формат YYYYMMDD)       |
| `en`        | `20260503`   | Конец недели (YYYYMMDD)               |
| `doctor`    | `990005142`  | ID врача                              |
| `filialId`  | `4`          | ID филиала                            |
| `speclist`  | `1031`       | ID специальности                      |
| `onlineMode`| `0`          | 0 = офлайн-запись, 1 = онлайн         |
| `_`         | `1776942758` | Timestamp-кешбастер (можно любой)     |

Сайт загружает расписание понедельно: одна неделя = один запрос.

### Формат ответа

```json
{
  "success": true,
  "data": [
    {
      "dname": "Фелицин Игорь Сергеевич",
      "freeCount": 0,
      "intervals": [
        {
          "workDate": "20260427",
          "startInterval": "10:00",
          "endInterval":   "13:40",
          "isFree":      false,
          "isAvailable": false,
          "filialName":  "АПО №5 по адресу пер.Днепровский,д.122/1",
          "schedident":  40199331
        }
      ]
    }
  ]
}
```

**Свободный слот:** `isFree: true` + `isAvailable: true`.  
**`freeCount`** — количество свободных слотов на неделю (быстрая проверка без перебора).  
Если `data: []` — врач в эту неделю не работает.

### Эндпоинт проверки сессии

```
GET https://er.center-zdorovie.ru/logged-in
```

Возвращает `{ "authenticated": true, ... }` если сессия жива.  
Скрипт вызывает его перед каждой серией проверок.

### Авторизация (куки)

Сайт работает на **Play Framework**. Используются три куки:

| Кука           | Назначение                                         |
|----------------|----------------------------------------------------|
| `PLAY_SESSION` | Идентификатор сессии Play Framework                |
| `WR_SESSION`   | Зашифрованная сессия медицинской системы WR        |
| `WR_DETAIL`    | Зашифрованные данные пользователя WR               |

Остальные куки (`_ym_uid`, `_ym_isad` и др.) — Яндекс.Метрика, для работы не нужны.

Сессия — **sliding**: сервер возвращает обновлённые куки через `set-cookie` в каждом ответе. Поэтому пока скрипт делает запросы хотя бы раз в 1–2 часа, куки живут вечно.

## Обязательные заголовки запроса

Без этих заголовков сервер может вернуть ошибку или редирект:

```
x-requested-with: XMLHttpRequest
x-integration-type: PORTAL
referer: https://er.center-zdorovie.ru/specialists
```

Они имитируют запрос, сделанный браузером через AJAX.
